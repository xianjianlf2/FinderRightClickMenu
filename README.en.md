# FinderRightClickMenu

English · [简体中文](README.md)

A small macOS Finder right-click menu utility: adds "Copy Path / Go to Enclosing Folder / Open Terminal Here" to the context menu.

- **MenuExtension**: the sandboxed Finder Sync extension. It builds the menu and reads the selected paths; actions that need extra privileges are handed to the companion app via a `frcm://` URL.
- **App**: the non-sandboxed companion app. It lives in the menu bar, receives `frcm://` to open a terminal / navigate / post notifications, and provides the "Permissions & Settings" window.
- **ThirdParty/PermissionFlow**: a vendored local Swift Package (MIT, sourced from capcap). Clicking the "Accessibility / Full Disk Access" slot in the settings window opens the matching System Settings pane and shows a floating panel that follows that window, guiding you to drag the app icon into the permission list.

## Build

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml` and is not checked in (signing and other local settings are excluded too).

```sh
brew install xcodegen        # if not installed
xcodegen generate            # generates FinderRightClickMenu.xcodeproj
xcodebuild -project FinderRightClickMenu.xcodeproj \
  -scheme FinderRightClickMenu -configuration Debug \
  -derivedDataPath build build
```

Configure signing to taste under Xcode's "Signing & Capabilities" (the default local ad-hoc signing is enough to run).

## Install

Copy the built `FinderRightClickMenu.app` to `/Applications` and open it, then enable
the extension under "System Settings → Privacy & Security → Extensions / Finder Extensions".

## Package a DMG / Release

```sh
scripts/release-dmg.sh 1.0.0              # build Release and produce dist/FinderRightClickMenu-1.0.0.dmg
PUBLISH=1 scripts/release-dmg.sh 1.0.0    # also publish a GitHub Release and upload the DMG via gh
```

This repo uses ad-hoc signing by default; it is not Developer ID signed or notarized, so a DMG
downloaded from GitHub is blocked by Gatekeeper on other Macs (see below). For proper public
distribution, configure a Developer ID certificate and notarize the artifact with `notarytool`
plus `stapler`.

## macOS verification block ("damaged / cannot be verified")

If macOS shows something like `Apple could not verify "FinderRightClickMenu" is free of malware`
or "is damaged and can't be opened", remove the quarantine attribute from the app bundle you
trust and reopen it:

```sh
xattr -dr com.apple.quarantine /Applications/FinderRightClickMenu.app
```

If you are running a local build rather than the copy in `/Applications`, replace the path with
the actual location, for example:

```sh
xattr -dr com.apple.quarantine ./build/Build/Products/Debug/FinderRightClickMenu.app
```

Only run this on builds you trust (a release downloaded from this repo or one you built locally).
After removing it, open the app normally, then enable the extension under
"System Settings → Privacy & Security → Extensions / Finder Extensions".
